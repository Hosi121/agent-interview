# Issue: 連絡先開示の求職者承認フローと企業単位の許可/拒否

## 背景
- 現状は採用担当者が一方的に連絡先開示できるため、求職者の信頼を損ねやすい
- 企業単位での公開可否を設定できると、供給側の心理的ハードルを下げられる

## ゴール
- 求職者が連絡先開示を承認/辞退できる
- 企業単位で「今後は自動許可/自動拒否」を選択できる

## 非ゴール
- 企業ブロックの全体設定画面
- 連絡先開示の返金・差分請求

## 要件
- 採用担当者は「連絡先開示リクエスト」を送る
- 求職者はリクエストに対して承認/辞退できる
- 承認時のみポイント消費される
- 求職者は企業単位で自動許可/自動拒否を設定できる
- 自動許可の場合はリクエスト時点で開示（ポイント残高がある場合）
- 自動拒否の場合はリクエスト時点で辞退確定

## 受け入れ基準
- CONTACT_REQUESTED の状態がUIとAPIで表現される
- 求職者が承認した時のみ CONTACT_DISCLOSED になる
- 企業単位の許可/拒否が保存される
- 自動許可/拒否が次回のリクエストに反映される

## テスト/確認
- 求職者承認・辞退のUIフロー確認
- ポイント消費のタイミングが承認時のみであること
- 自動許可/拒否設定の反映
