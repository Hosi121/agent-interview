# ビジネスモデル設計書

## 1. サービス概要

### コンセプト
「会話をした結果をAgentとして公開することで非同期な求職を実現する」

### 価値提案

| ステークホルダー | 価値 |
|-----------------|------|
| **求職者** | エージェントが24時間自分の代わりに対応。転職活動の時間的負担を軽減 |
| **採用担当者** | 候補者の人となりを事前に深く理解。面接効率の向上 |

### 非同期性の重要性
- 求職者は「公開したら待つだけ」
- 採用担当者との会話内容は求職者に見えない
- 採用担当者が「興味あり」と表明したときだけ通知

## 2. 課金モデル

### 2.1 基本構造：月額サブスク + 従量課金（ポイント制）

```
月額基本料金（ポイント含む）
    +
超過分のポイント追加購入
```

### 2.2 なぜこのモデルか

#### 月額のみ / 従量のみの問題

| モデル | 問題点 |
|--------|-------|
| 月額固定のみ | 使わない月も課金 → 解約リスク高。使いすぎても同額 → 機会損失 |
| 従量課金のみ | 収益不安定。顧客の予算管理困難。定着しにくい |

#### 月額+従量のメリット
- **事業者**: 最低収益確保 + ヘビーユーザーからの追加収益
- **顧客**: 予算内で計画的に利用可能

#### なぜポイント制か
1. **デカップリング効果**: 支払いと利用を分離し、利用時の心理的ハードルを下げる
2. **サンクコスト活用**: 「ポイントがあるから使おう」
3. **柔軟性**: アクションごとに異なるコスト設定が可能
4. **価格調整の容易さ**: ポイント単価の調整で対応可能

## 4. 競合分析と価格妥当性

### 4.1 競合サービスの価格帯

| サービス | モデル | 価格 | 特徴 |
|---------|-------|------|------|
| ビズリーチ | 月額+成果報酬 | 月額8〜15万 + 年収15% | ハイクラス |
| Wantedly | 月額 | 4〜15万/月 | カジュアル面談 |
| Green | 成果報酬 | 30〜90万/採用 | IT特化 |
| LinkedIn Recruiter | 月額 | 10〜20万/月 | グローバル |

### 4.2 ポジショニング

```
価格（高）
    │
    │  ビズリーチ    LinkedIn
    │
────┼──────────────────────────
    │                ★ 本サービス
    │  Green         (¥3〜20万/月)
    │  Wantedly
    │
価格（低）
    │
従来型 ─────────────────→ AI・非同期型
```

**「中価格帯 × 非同期AI活用」という空白地帯**

### 4.3 バリューベースの価格正当性

| 比較対象 | コスト | 本サービスの優位性 |
|---------|-------|------------------|
| 人材紹介（成功報酬） | 年収30% = 150〜300万円 | 月額数万円で候補者発掘可能 |
| 採用担当者の時間 | 時給3,000円 × 面接1h | 事前スクリーニングで工数削減 |
| スカウト返信率 | 平均5〜10% | エージェント会話で事前フィット確認 |

| **合計** | **¥10,000** | 顧客数で按分 |

### 5.2 顧客セグメント別シナリオ（現実的な想定）

#### 前提：「そんなに使われない」シナリオ

採用活動は常時行われるわけではなく、波がある。
多くの顧客は契約しても毎月フル活用しない。

---

#### ライトユーザー（全体の40%と想定）

**行動パターン**: 月に数回ログイン、様子見
```
会話:        5回/月   →  5pt
興味表明:    2件      →  0pt
連絡先開示:  1件      → 10pt
メッセージ:  2通      →  6pt
────────────────────────────
消費:        21pt / 100pt（21%消費、79pt余り）
APIコスト:   ¥50
```

#### 標準ユーザー（全体の40%と想定）

**行動パターン**: 週1-2回利用、継続的に候補者探し
```
会話:        20回/月  → 20pt
興味表明:    8件      →  0pt
連絡先開示:  3件      → 30pt
メッセージ:  10通     → 30pt
────────────────────────────
消費:        80pt / 100pt（80%消費、20pt余り）
APIコスト:   ¥200
```

#### ヘビーユーザー（全体の20%と想定）

**行動パターン**: 積極採用中、毎日利用
```
会話:        50回/月  → 50pt
興味表明:    20件     →  0pt
連絡先開示:  8件      → 80pt
メッセージ:  30通     → 90pt
────────────────────────────
消費:        220pt / 100pt（120pt超過）
追加購入:    120pt × ¥500 = ¥60,000
APIコスト:   ¥500
```


**APIコストが極めて低いため、利用率が低くても粗利は維持される**

- 解約率（Churn Rate）が上昇
- LTVが低下
- 口コミ・紹介が減少

## 6. リスクと対策

### 6.1 主要リスク

| リスク | 影響 | 対策 |
|-------|------|------|
| **利用率低下** | 解約増、LTV低下 | オンボーディング強化、利用促進メール |
| **ポイント余り** | 「損した感」→ 解約 | ポイント繰越、消費期限の設定 |
| **競合参入** | 価格競争 | AI品質での差別化、独自データ蓄積 |
| **求職者不足** | 採用担当者の価値低下 | 求職者向けマーケティング強化 |

### 6.2 ポイント余り問題への対策

| 対策 | 詳細 |
|------|------|
| **繰越制限** | 翌月への繰越は50%まで。使い切りインセンティブ |
| **消費期限** | 付与から3ヶ月で失効 |
| **ダウングレード推奨** | 利用率が低い顧客に下位プランを提案（解約より良い） |
| **利用促進** | 残ポイントが多い顧客へリマインドメール |

## 7. KPI設計

### 7.1 事業KPI

| KPI | 定義 | 目標 |
|-----|------|------|
| MRR | 月次経常収益 | - |
| ARR | 年次経常収益 | - |
| 顧客数 | 有料契約社数 | - |
| ARPU | MRR / 顧客数 | ¥50,000以上 |
| Churn Rate | 月次解約率 | 3%以下 |
| LTV | ARPU / Churn Rate | ¥1,500,000以上 |
| CAC | 顧客獲得コスト | LTV/3以下 |

### 7.2 プロダクトKPI

| KPI | 定義 | 健全な目安 |
|-----|------|-----------|
| DAU/MAU | 日次/月次アクティブ率 | 20%以上 |
| ポイント消費率 | 消費pt / 付与pt | 60%以上 |
| 会話→興味表明率 | 興味表明数 / 会話数 | 15%以上 |
| 興味表明→開示率 | 開示数 / 興味表明数 | 30%以上 |
| NPS | 推奨度スコア | 30以上 |
